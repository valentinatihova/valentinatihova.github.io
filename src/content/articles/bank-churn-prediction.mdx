## Problem framing

Banks optimize for retention when acquisition cost dominates. The task was crisp: from behavior and demographics,
predict **exit** soon, and push **F1 on a held-out test split** past a fixed gate (0.59 in the original rubric).
AUC-ROC stayed in the room as a sanity check on ranking quality, not as the optimization target.

## Data

The **Bank Customer Churn Modeling** dataset on Kaggle (10k rows, mixed numeric and categorical fields, binary
`Exited`) is public benchmark data — useful for comparing techniques without conflating results with any live portfolio.

## What constrained F1

### 1. Class imbalance vs. headline accuracy

With ~80% negatives, high accuracy can still mean useless churn detection. I compared **class weights**, **minority
upsampling**, and **threshold search on predicted probabilities**. **Balanced class weights inside a random forest**
won; upsampling moved some tree configurations but did not beat the best weighted forest on validation, and threshold
sweeps did not rescue models that never separated the minority class.

When the business metric is F1, tune for that objective end-to-end — including the decision threshold — but do not
expect thresholding to fix a model with no usable ranking signal.

### 2. Scaling without leakage

Numeric columns (`credit_score`, `age`, `balance`, `estimated_salary`, `tenure`, `num_of_products`) were standardized
with `StandardScaler` **fit only on the training slice** after a stratified 60 / 20 / 20 split. Geography and gender were
one-hot encoded with a consistent baseline drop to avoid redundant dummies.

Fitting the scaler on the full frame before splitting would leak aggregate statistics into validation and test — a
small mistake that quietly flatters metrics.

### 3. Weak linear signals, still informative

Correlation against `exited` put **age** and **balance** forward; `is_active_member` leaned negative in this slice
(active customers slightly more likely to churn) — a useful reminder that a single column is not a causal story
(product mix, geography, and competition sit behind the label).

![Schematic of Pearson correlations between selected features and the churn label](/articles/bank-churn/correlation-signals.svg)

### 4. Model families under imbalance

Unbalanced trees plateaued early; **logistic regression collapsed on F1** unless weights or resampling intervened.
Random forests with `class_weight='balanced'` and tuned depth / estimator count gave the best bias–variance trade for
this tabular problem.

Ensembles reduce variance, but imbalance still needs an explicit prior via weights or sampling — library defaults encode
a majority-class prior whether you name it or not.

### 5. Validation discipline

Hyperparameters were compared on the validation fold; the **test fold was reserved** for the final readout. The
reported run landed near **F1 ≈ 0.61** with **AUC-ROC ≈ 0.76** on test — above the 0.59 gate with margin.

## Tradeoffs I would carry to a production review

- **F1 vs. AUC-ROC:** AUC averages ranking across thresholds; F1 locks to one operating point. Stakeholders who pay for
  interventions usually care about the latter — report both, declare a primary metric before tuning.
- **Upsampling vs. weighting:** Upsampling duplicates information and costs training time; weighting keeps the empirical
  sample size honest. I still run both because some estimators interact oddly with `class_weight`.
- **Interpretability:** Shallow trees read well but cap performance; the weighted forest was the pragmatic compromise for
  this dataset size and noise level.

The durable output is methodological: a reproducible notebook, a single honest test touch, and a comparison narrative that
does not depend on one lucky split. No claim of lift on a real bank book — the value is **clean measurement hygiene**
